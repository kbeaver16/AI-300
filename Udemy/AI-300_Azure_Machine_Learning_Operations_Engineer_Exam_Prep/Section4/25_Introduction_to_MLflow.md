# MLflow — Introduction & Role in Azure ML

## Key Concepts

- **MLflow** — Open‑source platform for managing the ML lifecycle: tracking, model management, and deployment.
- **Experiment Tracking** — Logging parameters, metrics, artifacts, and source code for each run.
- **Model Registry** — Centralized store for versioned ML models.
- **Cross‑Platform Deployment** — MLflow models can be deployed in Azure ML, Databricks, and other MLflow‑compatible environments.
- **MLOps Lifecycle Integration** — MLflow supports experiment tracking, model versioning, and deployment workflows.

---

## Definitions

- **Run** — A single execution of training code that logs metrics, parameters, and artifacts.
- **Experiment** — A collection of related runs.
- **Artifacts** — Files generated during training (model binaries, conda.yaml, requirements.txt, input schema, etc.).
- **Model Signature** — Defines the expected input/output schema for a deployed model.
- **Managed Endpoint** — Azure ML real‑time inference endpoint (AKS, ACI, serverless, etc.).

## Step‑by‑Step Processes

### 1. Running Experiments with MLflow in Azure ML

1. Create an **experiment** in Azure ML.
2. Submit **jobs** (training runs) under that experiment.
3. MLflow automatically logs:
    - Parameters
    - Metrics
    - Model artifacts
    - Source code snapshot

### 2. Registering a Model

1. Navigate to the job’s **Outputs + Logs** section.
2. Locate the `model/` directory generated by MLflow.
3. Register the model in the **Model Registry**.
4. Assign version numbers and annotations.

### 3. Deploying a Model

1. Select the best model version from the registry.
2. Deploy to a **real‑time managed endpoint**:
    - AKS
    - ACI
    - Serverless
    - Managed compute instance
3. Use the `input_example.json` to build the request body for inference.

## Summary

MLflow is a core component of the Azure ML and Databricks MLOps ecosystem. It provides experiment tracking, model versioning, and cross‑platform deployment capabilities. Azure ML’s managed compute instances come preconfigured with MLflow, making it easy to log parameters, metrics, and artifacts. MLflow’s model format enables seamless deployment to Azure ML endpoints or Databricks, supporting a unified workflow across platforms.

---

## Actionable Takeaways

- Use MLflow to **standardize experiment tracking** across Azure ML and Databricks.
- Always log:
    - Hyperparameters
    - Metrics
    - Model artifacts
    - Input/output signature
- Use the MLflow‑generated `model/` directory for **model registration**.
- Leverage MLflow’s cross‑platform compatibility to avoid vendor lock‑in.
- Use the `input_example.json` file to quickly build inference payloads.
