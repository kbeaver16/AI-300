# AI-300 Study Repository

## Overview

This directory contains a comprehensive study plan and tracking materials for the **AI-300** exam (Microsoft Azure AI Engineer Associate). The repository is organized as a 6-week intensive study program covering MLOps infrastructure, machine learning lifecycle management, model deployment and operations, generative AI operations, and observability.

## Repository Structure

### Root Level Files

- **`00_Dashboard.md`** - Study progress tracker with overall status overview, module completion counts, and domain readiness checklist. Used to track progress across all 6 weeks.
- **`AI-300_Master.md`** - Master study plan with detailed weekly objectives, modules, concepts, hands-on labs, and checkpoints for all 6 weeks of study.

---

### Subdirectories

#### `01_StudyPlan/`
Contains the structured study curriculum and planning documents.
- **`StudyPlan.md`** - High-level study plan with timeline, learning resources, and study methodology.

#### `02_Daily/`
Daily study logs and progress tracking. Contains 35 daily study files (Day01.md through Day35.md), covering the full 6-week study period. Each file logs daily learning activities, concepts covered, and progress.

#### `03_Concepts/`
Conceptual reference materials and learning notes on key topics.
- **`AutoMLOverview.md`** - Comprehensive overview of Azure Automated Machine Learning
- **`AzureMLLearningPath.md`** - Learning path and resources for Azure Machine Learning
- **`EvaluateAutoMLExperimentResults.md`** - Guide on evaluating and interpreting AutoML experiment results

#### `04_Architectures/`
Reference architectures and architecture diagrams (currently empty, for future use).
Intended for documenting MLOps and GenAIOps architecture patterns explored during study.

#### `05_HandsOn/`
Hands-on lab exercises and implementation projects (currently empty, for future use).
Intended for hosting code, scripts, and lab outputs from practical exercises.

#### `06_Reference/`
Reference materials and supplementary documentation.
- **`Links.md`** - Curated links to Microsoft Learn modules, documentation, and external resources
- **`Snippets.md`** - Code snippets, command templates, and useful configurations for quick reference

#### `07_Templates/`
Reusable templates for creating study materials and documentation.
- **`ArchitectureTemplate.md`** - Template for documenting architecture diagrams
- **`ConceptTemplate.md`** - Template for concept notes
- **`Daily_Template.md`** - Template for daily study log entries
- **`HandsOnTemplate.md`** - Template for hands-on lab documentation

#### `Udemy/`
Supplementary materials and notes from Udemy course content (if applicable).

## Study Program Overview

### Duration
- **6 weeks** at 1.5-2 hours per day

### Learning Paths Covered
1. [MLOps Path](https://learn.microsoft.com/en-us/training/paths/operationalize-machinelearning-models/)
2. [GenAIOps Path](https://learn.microsoft.com/en-us/training/paths/operationalize-generativeai-applications/)

### Weekly Milestones

| Week | Focus | Objective |
|------|-------|-----------|
| 1 | MLOps Infrastructure | Design and implement an MLOps infrastructure |
| 2 | ML Training + Pipelines | Implement ML model lifecycle (training + orchestration) |
| 3 | Model Deployment + Operations | Complete ML lifecycle (deployment + monitoring) |
| 4 | GenAIOps Infrastructure | Design generative AI infrastructure |
| 5 | GenAI QA + Monitoring | Implement quality assurance and monitoring for generative AI |
| 6 | Optimization + Exam Prep | Performance optimization and final exam preparation |

## How to Use This Repository

1. **Start with the Dashboard** (`00_Dashboard.md`) to see overall progress and domain readiness
2. **Review the Master Plan** (`AI-300_Master.md`) for detailed weekly objectives
3. **Follow Daily Logs** (`02_Daily/`) to track daily learning activities
4. **Reference Concepts** (`03_Concepts/`) for in-depth explanations
5. **Use Templates** (`07_Templates/`) when creating new study materials
6. **Check Reference Materials** (`06_Reference/`) for links and code snippets
7. **Document Architecture & Hands-On Work** in `04_Architectures/` and `05_HandsOn/` as you progress
